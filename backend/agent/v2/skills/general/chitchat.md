## 技能概述

处理用户的闲聊、问候、情感表达等非任务性对话，提供温暖、自然的回应。

## 输入规范

| 参数 | 类型 | 必填 | 说明 |
|------|------|------|------|
| message | string | 是 | 用户原始消息 |
| emotion | string | 否 | 检测到的情绪状态 |

## 前置条件

- [ ] 无需特殊前置条件

## 执行逻辑

### 步骤1: 意图确认

确认用户意图是闲聊而非任务请求：
- 无明确任务关键词
- 无地点、时间、对象等实体
- 表达情感或进行社交互动

### 步骤2: 情绪识别

分析用户消息的情绪倾向：
- **积极**: 开心、兴奋、感谢
- **消极**: 难过、焦虑、抱怨
- **中性**: 问候、闲谈、好奇

### 步骤3: 回复策略选择

根据情绪选择回复策略：
- 积极情绪 → 共情+引导
- 消极情绪 → 安慰+支持
- 中性情绪 → 自然+引导

### 步骤4: 生成回复

生成符合以下要求的回复：
- 语言自然，符合对话场景
- 长度适中，不超过3句话
- 可适当引导至业务场景

## 约束条件

1. **长度限制**: 回复不超过100字
2. **语气要求**: 温暖、友好、不机械
3. **边界控制**: 不涉及敏感话题
4. **引导适度**: 不过度推销业务

## 输出格式

```json
{
  "status": "success",
  "data": {
    "reply": "回复内容",
    "emotion_detected": "detected_emotion",
    "should_continue": true,
    "suggested_topics": ["餐厅推荐", "电影推荐"]
  },
  "message": "回复内容"
}
```

## 对话示例

**用户**: 你好呀

**输出**: 你好！有什么可以帮你的吗？想找美食、看电影，还是聊聊？😊

---

**用户**: 今天好无聊啊

**输出**: 无聊的时候最适合探索新东西啦！要不我给你推荐附近新开的餐厅，或者找部好看的电影？
